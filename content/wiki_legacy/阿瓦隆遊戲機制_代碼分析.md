# 阿瓦隆遊戲機制代碼分析

## 系統架構概述

1. 本系統整合 Line, Discord 與 Google Sheets 構建阿瓦隆 (Avalon) 遊戲生態。
2. 核心邏輯包含訊息同步、戰績記錄與自動化分析。
3. 採用 Python `gspread`, `discord.py`, `line_notify` 為主要技術棧。

## 核心模組功能解析

### 聊天室橋接模組

1. **模組名稱**：`Avalon.py`, `utilities.py`, `discord_bot.py`, `line_bot.py`
2. **功能定義**：
   - **Line Webhook**：`line_bot.py` 基於 Flask 框架接收 Line Event，處理 Sticker, Image, Video 等多媒體訊息。
   - **訊息佇列**：實作 `_PENDING` 與 `_LOCKS` 機制，確保多線程下訊息順序發送。
   - **Discord 整合**：`discord_bot.py` 使用 `discord.ext.commands` 建立機器人，監聽指令 (`!`) 與事件。

   - **訊息同步**：使用 `zmq` (ZeroMQ) 進行 Python 程序間通信，實現跨平台訊息轉發。
   - **影音處理**：`utilities.py` 整合 `moviepy` 與 `pydub`，支援遊戲錄影之剪輯與音訊處理。
   - **外部資訊整合**：`lemonight/crawler.py` 實作 CNA 新聞 (`get_news`) 與伊莉電影 (`get_eyny_movie`) 爬蟲，增強機器人互動功能。

### 數據分析模組

1. **模組名稱**：`analysis.py`
2. **功能定義**：
   - **Google Sheets 整合**：使用 `gspread` 與 `oauth2client` 認證，連結 `avalonpediatw` 試算表。
   - **戰績存取**：自動化讀寫遊戲數據，支援 `pandas` 與 Google Sheets 間之資料交換。
   - **路徑管理**：動態偵測執行環境 (Local vs Github Action)，自動切換 `folder_path`。

### 遊戲邏輯層

1. **模組名稱**：`Avalon.py`
2. **功能定義**：
   - **狀態管理**：監控遊戲進程與玩家狀態。
   - **自動化排程**：整合 `time` 與 `datetime` 模組，執行定時任務。

## 零式規則檢核

1. **source_experience**: 整合自 Iteration 23 & 24 深度閱讀之 `結果產出` 目錄檔案。
2. **takeaway_principle**: 跨平台社群機器人需建立在穩定之 API 串接與錯誤處理機制上。
3. **model_used**: 採用 Serverless 概念與 Google Sheets 作為輕量化資料庫 (NoSQL-like)。
4. **evidence**: `analysis.py` 中 `get_gs` 函數之實作細節。
5. **next_test**: 驗證多線程下之訊息同步延遲與 API Rate Limit 限制。
