## Proposed schema (already implemented)

- Output = one YAML file per sheet under `content/_data/`.
- First non-empty row → headers (deduped on collision with `_N` suffix).
- Each subsequent row → dict keyed by headers; fully-empty rows dropped.
- Special-sheet aliasing via `SPECIAL_SHEETS`: `角色`→`roles`, `陣容`→`team_composition`, `規則`→`rules`.
- Merged-cell anchors propagated via `openpyxl` pass (graceful-degrade if pivot-cache bug hits).
- `_parse_summary.yaml` manifest: source path, generated_at (+08), per-sheet {rows, skipped, header_cols, bytes}.
- Type normalization: floats with integer value → int; pandas Timestamp / datetime → ISO8601; strings trimmed; NaN → None.

## Prototype snippet (core loop)

```python
xl = pd.ExcelFile(xlsx, engine="openpyxl")
for sheet_name in xl.sheet_names:
    df = xl.parse(sheet_name, header=None, dtype=object)
    df = fill_merged_ranges(xlsx, sheet_name, df, log)  # openpyxl merged-cell pass
    records, skipped, hdr_cols = sheet_to_records(df)   # headers + dict rows
    out_name = resolve_output_name(sheet_name, used_names)

```

## Known quirks / fragilities

1. **openpyxl + Python 3.14 pivot-cache bug** — workbook has pivot tables; openpyxl crashes on full load. Worked around by using pandas (which skips pivot machinery) for the main read, and calling openpyxl only for the merged-cell pass wrapped in try/except.
2. **Duplicate/blank headers** — handled via `unique_headers` (None-skip + `_N` dedupe), but downstream Astro collections need schema validation per sheet, not generic dicts.
3. **Sheet `1-1` is empty** — benign; emits near-empty YAML. Consider skipping in a future pass.
4. **Large sheets** — `牌譜` = 3117×48 → ~3MB YAML. Fine for build-time, but should be gitignored or split if it grows; keep raw xlsx out of repo.
5. **Input path is hard-coded to `E:/`** — external drive, non-portable. For CI/deploy, must either stage the xlsx into the repo under `data/master.xlsx` or pull from Google Drive.
