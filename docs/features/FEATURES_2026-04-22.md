## 1. Multi-Account Binding

**What**: Link your Discord, LINE, and Google accounts to a single Avalon profile.

### Why This Matters

- Play on any platform without creating separate accounts
- All game records, ELO, badges, and friends follow you
- Automatic merge if you already have an account under a different login method

### How to Use

1. **Access Profile**: Home → Stats → click "Profile Info" button
2. **Linked Accounts section**: Shows three rows:
   - Discord (with linked ID if connected)
   - LINE (with linked ID if connected)
   - Google (with email if connected)
3. **Bind a new account**:
   - Click "Bind" on the unlinked provider
   - You'll be redirected to login with that platform
   - After successful login, return to profile automatically
   - If that account already existed: game records + ELO + badges merge into your current profile
4. **Unlink an account**:
   - Click "Unlink" next to the provider
   - **Safety rule**: Must have at least one login method remaining
   - If it's your last one, button will be disabled

### Account Merge Rules

When binding to an account that already existed:

| Field | Merge Logic |
|-------|-------------|
| Game records | Both counts preserved (no loss) |
| ELO rating | Kept the higher rating (conservative) |
| Badges | Combined (all badges you earned) |
| Friends list | All friends from both accounts kept |
| Profile data | Filled from existing account where missing |

**Example**: 
- Your Google account: 100 games, ELO 1500, 3 badges, 5 friends
- Your Discord (already registered): 50 games, ELO 1600, 2 badges, 3 friends
- After binding Discord to Google: 150 games, ELO 1600, 5 badges, 8 friends

### Languages

- English: "Linked Accounts", "Bind", "Unlink", "Already linked"
- Traditional Chinese: "已綁定帳號", "綁定", "解除綁定", "已綁定"

## 2. Player Short Code

**What**: Every player gets an 8-character short code for easy identification.

**Format**: Alphanumeric (A-Z, 2-9) — excludes confusing characters (0/O, 1/I/L)

**Examples**: `7K3M9P2Q`, `VXMKN4HT`, `2R9FGQVN`

### How to Use

1. **View your code**: Home → Stats or Profile page
   - Displayed in gold, under your username
   - One-click "Copy" button
2. **Share with friends**: 
   - "My code is 7K3M9P2Q — add me!"
   - Easier than saying "my UUID is d4c3b2a1-..." on voice chat
3. **Quick add**: Friends page → Search tab
   - Paste the short code in search field
   - Click "Add Friend"
   - Instant follow without typing full username

### Data

- Auto-generated on registration
- Unique per player (system prevents duplicates)
- Existing players get auto-generated on first profile page visit
- Stored in database, visible forever (not regenerated)

## 3. Public Lobby Chat

**What**: Real-time chat room visible on the home page.

### Who Can Participate

| User Type | Can Send | Can View | Notes |
|-----------|----------|----------|-------|
| Registered (any provider) | Yes | Yes | 200 char limit, 2 msgs/2sec rate limit |
| Guest | No | Yes | Locked input with message: "Register to chat" |

### How to Use

1. **Home page**: Right panel shows "Lobby Chat"
2. **Empty state**: "No messages yet — say hello!"
3. **Send message**: Type in input box, press Enter or click Send
   - Max 200 characters
   - Message appears immediately on your screen
   - Instantly delivered to all online users
4. **Message bubble**: 
   - Your messages: Right-aligned, blue background
   - Others' messages: Left-aligned, gray background
   - System messages: Centered, e.g., "Alice joined the lobby"
5. **Guest mode**: 
   - Locked icon next to input
   - Text: "Guests can view but not chat. Register or sign in to participate."

### Server Behavior

- Keeps 50 most recent messages in memory
- Server restart clears chat (messages not persisted to DB in Phase 2)
- Future phases: Will be persisted to database

### Accessibility

- Full i18n: English + Traditional Chinese
- Accessible via keyboard: Tab to input, Enter to send
- Mobile-responsive: Works on iOS/Android

## 4. Cross-Platform Chat Mirroring

**What**: Lobby chat automatically syncs outbound to LINE group and Discord channel.

### Supported Platforms (Outbound)

- **LINE**: Designated group (configured by admin)
- **Discord**: Designated channel (configured by admin)

### How It Works

**Outbound** (Lobby → External):
1. User sends message in lobby chat: "Looking for a game!"
2. System automatically posts to LINE group: 
   ```
   [Avalon] Alice: Looking for a game!
   ```
3. Same message posted to Discord channel
4. User doesn't need to do anything — automatic

**Inbound** (External → Lobby): Coming in Phase 3

### Rules & Safety

| Feature | Behavior |
|---------|----------|
| Message format | `[Avalon] PlayerName: message` |
| Length limit | 200 characters (external platforms trim with `…`) |
| Rate limit | 5 messages/min per external user (anti-spam) |
| Loop prevention | If message came from LINE/Discord, it doesn't re-sync (prevents infinite loops) |
| Error handling | Fails silently — external platform down won't break lobby |
| Account mapping | TBD Phase 3 (future: bind external users to Avalon accounts) |

### Admin Setup

Requires:
1. Create public LINE group, invite Avalon bot, note group ID
2. Create Discord channel, invite Avalon bot, note channel ID
3. Provide IDs to deployment config (env vars):
   ```
   LOBBY_MIRROR_LINE_GROUP_ID=C123abc...
   LOBBY_MIRROR_DISCORD_CHANNEL_ID=987654321
   ```
4. If not configured: System logs info message, chat works normally (no external sync)

## 5. 9-Player Variant Enhancements

**What**: Full support for 9-player games with all special role combinations.

### Game Modes

#### Standard 9p (Oberon Optional)

- **Base setup**: 6 Good + 3 Evil
- **New**: Toggle Oberon
  - OFF: Keep 6 Good / 3 Evil (traditional Avalon rules)
  - ON: Convert 1 Good to Oberon → 5 Good / 4 Evil
- **Quest rounds**: Always 3-4-4-5-5 (unchanged)
- **Available roles**: Merlin, Percival, Morgana, Mordred, Oberon, Loyal Servant, Assassin (all freely combinable)

#### 9p Oberon Forced Mode

- **Fixed setup**: 5 Good + 4 Evil (Oberon always present)
- **Quest rounds**: 4-3-4-5-5 (different from standard)
- **Used for**: Oberon-heavy play style testing

### How to Create

1. **Create Game** → Select "9 Players"
2. **Choose Mode**:
   - Standard: Toggle "Include Oberon?" ON/OFF
   - Forced: Select "Oberon Forced" if you want specific rounds
3. **Role Selection**:
   - All role toggles independent (Percival, Morgana, Mordred, Oberon)
   - No restrictions (unlike previous Phase 1)
4. **Start**: When 9 players join, game begins normally

### Default Behavior

- **New games** default: Oberon ON (5 Good / 4 Evil)
- To use classic 6/3: Create game, explicitly toggle Oberon OFF

### Technical Details

- Role assignment engine handles swaps internally
- No new characters added (uses existing role set)
- Lady of the Lake mechanics unchanged
- All existing 7p/8p/10p+ modes unaffected

## 7. ELO System (Three-Phase Attribution)

**What**: Skill rating system with game outcome attribution.

### Phases

#### Phase 1: Global Prior (Now)
- Win/loss on game outcome (win = +ELO, loss = -ELO)
- Base delta: ~15-30 points per game (depends on opponent rating)
- Factor: Role, game size, team composition
- Data source: Historical games (2145+ records)

#### Phase 2: Role Granularity (Coming)
- Per-role attribution (Merlin earns different delta than Assassin)
- Outcome-specific (e.g., "Good player but Merlin got assassinated" = partial credit)
- Smoother calibration for casual vs. hardcore players

#### Phase 3: Per-Player Personas (Coming)
- Individual learning curve (badge/player recognition)
- Playable "personas" of top players
- Advanced analytics dashboard

### How to View

**Profile page → Stats section:**
- Current ELO rating (top)
- ELO change per recent game (green +/red -)
- Chart: ELO trend over last 50 games
- Breakdown by role (which role gives best ELO?)

### Calculation

(Simplified Phase 1 example)
```
base_delta = 16 * (outcome - expected_win_probability)
- outcome: 1.0 (win), 0.0 (loss)
- expected_win_probability: ~0.5 if similar ELO, <0.5 if higher-rated opponent, >0.5 if lower-rated
- Adjust for role (Merlin weight +20%, Assassin weight -10%, etc.)
```

### Safety

- Minimum game count to affect ELO: 5 games (avoid noise)
- Maximum delta per game: ±50 points (prevents spiking)
- Glicko 2 NOT used yet (defer to Phase 3)

## 8. Historical Data Learning for AI

**What**: AI opponents learn strategy from 2145+ real games.

### What AI Learns

1. **Team Formation** (selectTeam):
   - How often good players pick themselves (93% baseline)
   - How often evil players hide (varies by role)
   - Who gets picked first/last

2. **Voting** (voteOnTeam):
   - Good team approval rate: ~38% round 1, ~52% later
   - Evil team rejection likelihood: ~70% (from historical data)
   - Stricter standards when teammate already failed

3. **Quest Voting** (voteOnQuest):
   - Evil failure rate: 55-60% early, 85-90% when losing (pressure)
   - Good approval rate: ~95% (trust teammate)

4. **Lady of the Lake**:
   - Merlin targets Loyal 48% of time, Percival 19% (covers all bases)
   - Evil targets Good 55% of time (information gathering)

### Impact on Gameplay

- **Smoother**: AI feels less robotic, more human-like
- **Balanced**: Self-play testing shows good/evil win rates in expected range (30-40% good, 60-70% evil)
- **Transparent**: All decision constants logged, rollback available if needed

### Feature Flag

- `USE_HISTORICAL_PRIOR`: Environment variable (default: true)
- If disabled: Reverts to Phase 1 hardcoded strategy (always available as fallback)

### Data Privacy

- Historical games anonymized (no player names in game data)
- AI learning happens offline (no personal data queried at runtime)
- Players' real game decisions not exposed to others

## 11. FAQ

### Q: I linked my Discord account but my old games didn't appear.
**A**: 
1. Ensure you're signed in with the same Discord account
2. Check that you linked to the correct profile (not a new one)
3. Contact support with your short code and Discord ID

### Q: Can I bind the same Discord account to multiple Avalon profiles?
**A**: No — one Discord account can only link to one Avalon profile. If you want to switch, unlink first, then link to another profile.

### Q: My lobby chat isn't showing messages from LINE.
**A**: Phase 2 only sends Avalon messages to LINE (outbound). Inbound sync (LINE → Avalon) comes in Phase 3.

### Q: How often is my ELO updated?
**A**: After every game you play. You'll see the change on your profile immediately.

### Q: What's the minimum ELO to join a high-level game?
**A**: No ELO gate currently. Everyone can join any room. ELO is for tracking only (Phase 3 may add ranked mode).

### Q: Why did my short code change?
**A**: It shouldn't. Short codes are permanent. If you see a different code, you may be logged into a different account. Check your linked accounts.
