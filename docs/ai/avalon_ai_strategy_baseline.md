## §0.0 投票符號規則（Vote Token Encoding）

### Edward verbatim（2026-04-22 14:18）
> 沒有特別標示 代表正常票 (被選=場內投白, 沒被選=場外投黑)
> 有特別標示+- 代表異常票 (沒被選=場外投白, 被選=場內投黑)

### 解析規則
- **無標** → 正常票:in_team ? approve : reject
- **+** → 異常白:沒被選卻投 approve
- **-** → 異常黑:被選卻投 reject

### 戰術意義
- 無標 = 直覺票（資訊量低）
- + = 反直覺 approve（放水 / 混隊、紅方常見）
- - = 反直覺 reject（自我否定 / 洗白、派西偶用）

**此規則適用於所有歷史牌譜解析。基於此規則聚合 action_priors.json 與 TOP10 priors。**

**替代舊解讀（作廢）**:之前 subagent 自行推斷的「+ = approve / - = reject / 無標 = 跟多數」錯誤，相關資料檔已重跑。

## §0. 聽牌規則（最高優先硬規則,overrides 其他所有 quest action 決策）

**Edward 2026-04-22 12:38 +08 原話**:
> 不管是紅藍哪一方, 只要有一方聽牌(藍方已拿兩局藍 or 紅方已拿兩局紅), 那紅方都該只考慮"先讓任務失敗"
> 因為 三紅 => 紅方直接獲勝
> 三藍雖然進入刺殺環節, 但依然可能會刺錯
> 所以紅方聽牌(紅方已拿下兩局), 能讓任務失敗必失敗
> 藍方聽牌(藍方已拿下兩局), 同樣也是能讓任務失敗必失敗 (否則會直接進入刺殺環節)

Edward 2026-04-22 12:42 +08 強化:
> 紅:藍 = 2:0 的時候 紅方能拿下任務就該直接任務失敗

### §0.1 聽牌定義

**聽牌** = 任一方已贏 2 局(即 `goodWins === 2` 或 `evilWins === 2`)。

### §0.2 紅方 quest action 決策樹（聽牌觸發時）

```
IF 聽牌觸發(goodWins === 2 || evilWins === 2) AND 紅方在隊:
  → quest_vote = fail  (能失敗必失敗)

ELSE:
  → 走其他邏輯(TOP10 情境化、角色差異化、奧伯倫例外)
```

### §0.3 理由（為什麼聽牌兩端都 fail）

**紅方聽牌（evilWins === 2）**:
- 再拿 1 局 → 紅方任務路線直接獲勝,**不用**進刺殺環節。
- 放過此局 = 把機會讓回給藍方 = 多 1 場刺殺不命中的風險。

**藍方聽牌（goodWins === 2）**:
- 放過此局 → 藍方 3 任務完成 → 進入刺殺環節 → 刺殺不保證命中。
- 阻擋此局（fail）→ 比分 2-1 → 紅方繼續有機會反拿兩局(2-2、紅方 3 局)或再拖一輪。
- **期望值角度**: fail 這局 > 賭刺殺命中。

### §0.4 例外

唯一例外: **奧伯倫(Oberon)**。奧伯倫無隊友協調資訊,legacy 隨機行為保留(參 §3.2、§6.14),不套聽牌鐵則。其他紅方角色（刺客/莫甘娜/莫德雷德）聽牌場景一律 fail。

### §0.5 覆蓋舊邏輯

- 本規則 overrides §2「裝忠賭刺殺」路線在 `goodWins === 2` 場景的任何「放水」建議 — 藍方聽牌時**不能放水**。
- 本規則 overrides §6.1「紅方 2-0 領先時固定高機率出失敗忽略刺殺路線」在 `evilWins === 2` 的分析 — 紅方聽牌時反而**必 fail**(再拿一局直接贏,根本不用進刺殺階段)。
- 角色差異化(§6.14、Fix #5)只在**非聽牌**(兩邊都 < 2 局)場景套用;聽牌一觸發,所有紅方角色(除奧伯倫)都是 fail。

## §1. 陣營勝利條件（雙勝利，非單勝利）

Avalon **不是**「三任務定生死」遊戲。兩條勝利路線同時存在,**紅方可以用任一條贏**:

### 1.1 主路線（任務計分）

- **藍方贏任務**: 5 任務中 3 任務成功（由參與隊伍的玩家黑白球決定;**好人必投成功**,**紅方可選成功/失敗**）
- **紅方贏任務**: 3 任務失敗(1 任務只需 1 顆失敗,7+ 人局的第 4 任務例外需 2 顆) OR 任一輪第 5 次投票仍未過
- **結論**: 紅方 `紅方失敗票數 × 時機` 直接轉勝;藍方只能「擋掉 3 個失敗」

### 1.2 副路線（刺客 vs 梅林 / 刺殺翻盤）

- 藍方完成 3 任務 → 進入刺殺階段
- 刺客從**非刺殺陣營**玩家中指認一人為梅林;**刺中 = 紅方翻盤贏,刺錯 = 藍方贏**
- 多數局（約 **35% - 45%**,資料面 2145 場對應）的紅方是**透過刺殺**而非任務全失敗贏的

### 1.3 **關鍵不對稱**

| 項目 | 紅方 | 藍方 |
|---|---|---|
| 勝利路線數 | 2（任務 + 刺殺）| 1（擋任務 + 保梅林） |
| 任務輪內動作 | 可成功可失敗（戰術選擇） | 必成功（無選擇） |
| 資訊結構 | 互相認識 + 有「藏身」壓力 | 隱藏身分 + 要「辨識」壓力 |
| 失敗成本 | 紅方失敗 = 暴露 | 藍方「錯信」= 梅林暴露 |

## §2. 雙重路線策略:單純追任務 vs 裝忠賭刺殺

### 2.1 紅方三種路線（不是只有「追任務失敗」一條）

| 路線 | 條件 | 動作描述 |
|---|---|---|
| **A. 任務全失敗** | 紅方可穩定混進 3 輪任務 | 每輪進隊必 fail,吃掉 3 任務 |
| **B. 裝忠保刺殺** | 藍方任務壓力大,紅方先放水 1-2 任務 | 前期 fail-rate 低、偽裝好人、後期準確刺梅林 |
| **C. 混合** | 已拿 1-2 紅,但被抓得兇 | 下 1-2 任務改出成功、收斂暴露、留刺殺壓箱 |

### 2.2 AI 必須能分辨的關鍵變數

- **目前局勢**: 藍 0-紅 0 / 藍 1-紅 0 / 藍 2-紅 0 / 紅 1-藍 1 / 藍 2-紅 1 / 紅 2-藍 1 / 紅 2-藍 2 ...
- **當前玩家身分**: 刺客（優先藏,賭刺殺)/ 莫德雷德（隱藏性極高,可多放水)/ 莫甘娜（Percival 看得到,藏身度低)/ 奧伯倫（單兵作戰,高機率失敗票)
- **當輪進隊壓力**: 有沒有被推上來?團隊是否已有另一紅?
- **失敗票需求數**: 7+ 人局第 4 任務需 2 顆失敗 → 1 顆不夠用時單獨出失敗 = 浪費 + 暴露

### 2.3 「新手錯誤」級基準

紅方**不應**「只要進隊就出失敗」。AI 若用單一固定機率（例如固定 60% fail)忽略上述所有變數 → **連新手都不該犯的錯**(見 §6.1)。

## §3. 7 角色個別目標(AI 必須明確分辨)

### 3.1 好人陣營（藍方,3-5 人）

| 角色 | 私人資訊 | 核心目標 | 不能做的事 |
|---|---|---|---|
| **梅林** | 看得到紅方（除莫德雷德、奧伯倫) | 暗中引導好人排除紅方,**但不能暴露自己** | 明目張膽黑紅方(讓刺客看出)、派隊只派全好(明示自己有資訊) |
| **派西維爾** | 看得到梅林 + 莫甘娜(不知誰是誰) | 保護梅林(賭對的拇指是梅林)、吸引刺客注意自己 | 跳派(宣稱自己是派西)、沒看派票就選一根 |
| **忠臣** | 只知自己是好人 | 靠觀察派票、黑白球、任務結果推斷紅方;**保護可能的能力者**；派票派「你覺得的好人」| 隨便開場外白(內白外黑無意義)、自暴判斷邏輯 |

### 3.2 壞人陣營（紅方,2-4 人）

| 角色 | 私人資訊 | 核心目標 | 特殊規則 |
|---|---|---|---|
| **刺客** | 認識其他紅方（奧伯倫除外) | 藏身 + 準確抓梅林 | 最終刺殺權在此 |
| **莫甘娜** | 認識其他紅方（奧伯倫除外) | **對派西假裝是梅林**,誘導派西保錯人 | 派西看得到我 → 要模仿梅林行為 |
| **莫德雷德** | 認識其他紅方（奧伯倫除外) | **梅林看不到我** → 最高隱蔽性 → 可高頻進隊 + 多放水裝忠 | 梅林視角盲區,可深度潛伏 |
| **奧伯倫** | **不認識任何紅方** | 單兵作戰;進隊幾乎必投失敗(因為無隊友協調) | 紅方視角:奧伯倫是「隨機變數」,不能指望配合 |

### 3.3 AI 必備判斷規則（若任一項缺失 = 連新手都不該犯的錯）

1. 梅林派票**永遠不能把紅方放進去**(wiki:「梅林派票不要派紅方」)
2. 派西**不能直接選拇指**,要先看派票（wiki:「對梅林的基本尊重」)
3. 刺客選擇刺殺目標時,必須**排除一直派拇指的玩家**(派拇指是派西行為,非梅林)
4. 莫德雷德存在時,梅林的 `knownEvils` 清單不能含莫德雷德（code 已實作`SelfPlayEngine.ts:305-311`)
5. 奧伯倫不能出現在紅方 `knownEvils` 中（從紅方角度);但仍屬 evil team(code 已實作 `SelfPlayEngine.ts:322-324`)

## §4. 湖中女神 (Lady of the Lake) 互動規則

### 4.1 遊戲機制(canonical 7+ 人局預設開啟,見 `SelfPlayEngine.ts:157-186`)

- 第 2、3、4 任務結束後,**湖中女神持有者**選擇一位其他玩家(未曾被湖過的),**看到對方真實陣營(好/壞,看不到角色)**
- 湖完後,湖中女神**轉交給被湖的玩家**
- 結果**對持有者私人**,其他人只看到「湖過誰」不看結果 → **資訊不對稱武器**

### 4.2 持有者 / 目標動機矩陣

| 持有者陣營 | 目標陣營 | 持有者的理性動作 |
|---|---|---|
| 好人 | 嫌疑紅 | 湖過確認 → 下輪投票/派票可用此私人資訊 |
| 好人 | 嫌疑藍 | 浪費一次湖；除非想證明特定好人清白 |
| 紅方 | 嫌疑紅（已知隊友） | **可湖並宣告好人**(洗白隊友,把壓力轉移到好人)— Edward 2026-04-22 12:39 verbatim「紅方湖中女神當然可以湖隊友並宣告隊友是好人 / 不用刻意避開」 |
| 紅方 | 嫌疑藍 | 湖完仍結果誠實(好人) → 紅方私人資訊「這是藍」；若說謊稱「這是紅」 → 誤導好人 |
| 紅方(奧伯倫) | 任意 | 高不確定性;湖完只能得出「這是好/壞」,仍不知道是梅林/派西 |

### 4.3 AI 必備規則

1. **好人不能湖自己已知陣營的人**(不能湖 `knownEvils` 上的好人隊友 — 浪費);**紅方可以湖隊友**(Edward 2026-04-22 12:39 verbatim),策略上優先湖**公開歷史上明顯被懷疑的隊友**並宣告 good 以洗白
2. **好人持有者優先湖最可疑的**(高 suspicion 分數);紅方持有者在「洗白隊友」與「確認不知陣營藍方」之間權衡
3. 湖中結果**會影響後續派票 / 投票判斷**(不能湖完不做事)
4. 湖中結果**是否說謊**是紅方策略變數 → AI 需能模擬「說謊 or 誠實」的權衡

### 4.4 當前 code 現狀

- `SelfPlayEngine.ts:157-186`: 好人選第一個 `knownEvils`(若未被湖過);紅方 **純隨機**
- `HeuristicAgent` 沒有 `LadyOfTheLakeAction` 動作類型(`types.ts:63-67` 只有 4 類)
- 湖中結果**沒有**反饋到 suspicion map（ingest 函數看不到湖中事件）
- **連新手都不該犯的錯(見 §6.9)**: 紅方湖純隨機 = 湖到自己隊友 = 浪費

## §5. 提案 / 投票 / 任務三階段基本判準

### 5.1 提案(隊長選人,`team_select`)

**新手判斷**(AI 必須做到):
- 好人隊長:自己 + 低 suspicion 前 N-1 個;**不能含已知紅方**
- 紅方隊長:自己 + (50% 機率)一個隊友 + 填補好看的位子

**進階思考**(Phase 2+):
- **派票順序暴露身分**: 隊長在局面 1-2 可派出「反邏輯組合」製造矛盾(見 `/YhEMfFyMQlugT7AZNvxrIg` 外白球 wiki)
- **填空派票**: 為了特定驗證目的(例如驗證某玩家是不是派西)故意做非最優派法
- **下一任隊長是誰**: 當前局面是否要讓對手下任隊長也派得出合理隊?(座次壓力)

### 5.2 投票(黑白球,`team_vote`)

**新手判斷**(AI 必須做到):
- 好人 + 已知紅方在隊 → **必 reject**
- 好人 + 隊伍含失敗過的玩家 → reject
- 好人 + 第 5 次投票 → **force approve**(避免直接送任務給紅方)
- 紅方 + 自己或隊友在隊 → approve
- 紅方 + 全藍隊 → **不能全都 reject**(5 次 reject = 送局)

**進階思考**(Phase 2+):
- 外白(場外白球):不在隊的好人投 approve → **必須能說明為何**,不然是「新手亂開外白」(見 §6.7)
- 內黑(場內黑球):在隊的好人投 reject → 同上,必須有合理解釋(某鐵壞在隊,但未被隊長抓出)
- 尾家票(最後一個投的人):資訊最大 → 投票門檻應最嚴

### 5.3 任務黑白球(`quest_vote`)

**新手判斷**(AI 必須做到):
- 好人 → **永遠成功**(無選擇)
- 紅方:根據上述§2 的路線選擇成功/失敗

**進階思考**(Phase 2+):
- 紅 2-藍 0 時,紅方**不一定要**全員出失敗 → 若已贏在望但可以**藏更深**為刺殺鋪路,可以**放水成功**
- 紅 0-藍 2 時,進隊的紅方**幾乎必須出失敗**(再輸就沒機會了)— **除非**這紅方是梅林的強候選,出失敗會直接曝露 + 死
- 7+ 人局第 4 任務需 2 顆失敗 → 如果隊中**只有 1 個紅方**,單獨出失敗 = 浪費 + 直接暴露自己
- 奧伯倫進隊 → **高機率必失敗**(不知道是否隊友在內協調),難以 Phase B 類型的「放水留刺殺」戰術

## §6. 新手不應犯的錯清單

以下每條都是「連新手玩家(玩 10 場以上)都不會犯的錯」。**AI 若犯 = 連新手都不該犯的錯(Edward 2026-04-22 用語)**。

### §6.1 紅方 2-0 領先時**固定**高機率出失敗(忽略刺殺路線)

> **v1.1 更正**: §0 聽牌規則(2026-04-22 12:38 Edward 原話)**overrides** 本段原本的「放水賭刺殺」建議。
> 紅方聽牌(`evilWins === 2`)的正解是**必 fail** — 再拿 1 局紅方任務直接獲勝,根本不用進刺殺階段。
> 本段保留僅供歷史對照(記錄舊版 Fix #3 方向錯誤:紅方 2-0 裝忠放水),**實作以 §0 為準**。

**現象(歷史觀察)**: AI 在 `goodWins == 0 && evilWins == 2` 時用接近 100% 機率 quest_fail(其實是對的方向)。
**為什麼**(更新後):
- 紅方聽牌 → 再贏 1 局 = 紅方任務路線直接獲勝,不用進刺殺環節(風險更低)
- 舊版誤把「放水賭刺殺」當更優解,實際上**「再拿 1 局直接贏」> 「刺殺可能刺錯」**
- 對稱:藍方聽牌(`goodWins === 2`)紅方也必 fail — 不阻擋就進刺殺,刺不中就輸

### §6.2 好人投票不關心隊伍是否包含自己不信任的對象

**現象**: AI 計算平均 suspicion 時,不檢查「有沒有特定已知失敗過的隊員」就直接投票。
**為什麼新手不該犯**:
- 任務結果是**最高可信度資訊**(wiki 入門行動方針)。曾參與失敗任務的玩家 suspicion 應該**獨立於平均值直接觸發 reject**
- 現況:`HeuristicAgent.ts:344-346` 有 `hasFailedMember` 檢查,算 OK,但下面一段 L350-367(off-team 路徑)在沒有 fail member 時**仍用固定 baseline 機率投票** → 忽略「我看不順眼的隊長 + 一個 suspicion=2.5 的人」這種情境

### §6.3 梅林派票時把已知紅方放進去

**現象**: AI 以「看起來可信度高」排序挑人,不檢查是否踩到 `knownEvils`。
**為什麼新手不該犯**:
- 梅林**唯一功能**是資訊,**派紅方 = 等於告訴所有人「我沒看」** = 自暴身分 + 還把紅方送上任務
- Wiki 明示:「梅林派票不要派紅方」
- 現況:`HeuristicAgent.ts:268-289` 好人分支按 suspicion 排序,但**沒有明確保證** `knownEvils` 會排到最後;`knownEvils` 在 `onGameStart` 被設為 suspicion=10,間接地排到後面 → **間接正確但無硬保證**

### §6.4 派西維爾不區分梅林 / 莫甘娜 signal

**現象**: AI 持 `knownWizards = [M1, M2]` 但把兩個都當同樣可信度對待。
**為什麼新手不該犯**:
- 派西**唯一戰術價值** = 靠觀察兩個拇指的行為差異(派票、投票、上任務結果)來**賭對的那根**
- 現況:`HeuristicAgent.ts:275-286` 派西派隊時挑 `knownWizards[0]` 加入 → **等於隨機挑,沒分辨** → 派西與忠臣無功能差異

### §6.5 刺客不追蹤投票模式推測梅林

**現象**: AI 找梅林時只看「誰反對過含紅方的隊」或「誰從未上過失敗任務」,**不看投票 pattern 的一致性**。
**為什麼新手不該犯**:
- 梅林**最容易洩漏**的是投票模式(ref wiki「梅林/派西 資訊釋放品質」,為 ELO 四因子之一)
- 現況:`HeuristicAgent.ts:466-503` `getMerlinScore` 已有 hard mode 的 `teamHadEvil + rejection` 加權 + `allSucceeded quest appearances` 加權 → **部分做到**,但**沒考量**「梅林不應該派紅方」的派票行為(若玩家身為隊長從未派過紅方 → 高度梅林信號)

### §6.6 隊長提案時忽略座次順序(下一任隊長誰)

**現象**: AI 選隊只看當前 team composition,不考慮「如果被 reject,下一個隊長是誰 + 下一個隊長的可能提案」。
**為什麼新手不該犯**:
- 座次壓力是經典 Avalon 戰術(wiki「派票策略」整個章節)
- 若下一任隊長是強紅方,「送 reject」= 讓紅方上場 = 送局
- 現況:`HeuristicAgent` **完全沒有**座次相關邏輯;`SeatOrderAdjustment.ts` 只存在於 ELO 歸因層(已計算但沒餵回 agent 決策)

### §6.7 好人亂開場外白 / 內黑外白(沒 context 的判斷)

**現象**: AI 隨機機率決定是否 approve 不含自己的隊(例如 `OFF_TEAM_REJECT_BASELINE = 0.7` 固定值)。
**為什麼新手不該犯**:
- Wiki `為什麼不該亂開場外白球`:「開白球 = 公開宣稱贊成這個組合出任務 → 背後意義是『這個組合對我陣營有利』」
- 隨機固定 0.7 baseline = 沒 context 決策 = 開了沒意義的白/黑球
- 現況:`HeuristicAgent.ts:43-46` `OFF_TEAM_REJECT_BASELINE = { hard: 0.7, normal: 0.55 }` + L364 round_1 relax × 0.6 = **Edward 2026-04-22 06:26 點名的「60% 太低了」問題** → **這就是主問題**

### §6.8 忠臣派票時不考慮保護可能的能力者

**現象**: AI 好人一律「包自己 + 挑最低 suspicion 前 N-1 位」,不會有意識地**把可能的梅林 / 派西放進隊**以保護他們的資訊釋放。
**為什麼新手不該犯**:
- 入門行動方針wiki:「忠臣 = 任務派票就是選你覺得的『好人』」,但實戰上**能力者被派可以更多資訊流**
- 現況:`HeuristicAgent.ts:276-286` 派西分支**有**「至少加一個 `knownWizards` 進隊」邏輯,但**忠臣沒有對稱邏輯**(忠臣不知道誰是梅林,但能觀察哪個玩家「一直準確黑紅方」= 疑似梅林,應優先排入隊保護)

### §6.9 湖中女神持有者「湖自己已知陣營的人」= 浪費

**現象**: 紅方持湖隨機 pick target;好人持湖 pick `knownEvils[0]`(**但 `knownEvils[0]` 自己就知道,湖過沒新資訊**)。
**為什麼新手不該犯**:
- Wiki「湖女常見錯誤」:「鐵壞拿到湖中, 把湖女還給鐵好」/「鐵好拿到湖中, 湖另一個鐵好」= 經典爛操作
- 現況:`SelfPlayEngine.ts:167-172` 好人分支 **直接湖 `knownEvils[0]`** = **湖自己已知是紅方的人 = 新手錯誤最高級別**
- 正確:好人應湖**嫌疑最高的未知玩家**(非 `knownEvils`、非自己、非已湖過);紅方應湖**嫌疑最高的未知玩家**(非 `knownEvils`、非自己、非已湖過)

### §6.10 紅方隊伍只含 1 個紅方、需要 2 顆失敗時單獨出失敗

**現象**: AI 固定「紅方進隊 = 投失敗」,無視 `failsRequired == 2`。
**為什麼新手不該犯**:
- 7+ 人局第 4 任務的 2 顆失敗規則是 Avalon 核心變體;1 顆失敗 = 任務成功 + 那個紅方**直接暴露**(因為只有紅方會出失敗)
- 現況:`HeuristicAgent.ts:408-415` **有**處理 `failsRequired >= 2` 情境,算是對的 → **這點做到了,不算新手錯誤**

### §6.11 好人面對連續 4 次 reject 還繼續 reject(自動送局)

**現象**: AI 不知道「第 5 次 reject = 紅方自動贏」規則,繼續按邏輯 reject。
**為什麼新手不該犯**:
- 遊戲基本規則 = 送局,紅方免費贏
- 現況:`HeuristicAgent.ts:320-322` **有** `FORCE_APPROVE_FAIL_COUNT = 4` → **做到了,不算新手錯誤**

### §6.12 投票完全不紀錄 / 不從過去投票推斷身分

**現象**: AI 只看當前 state,不累積對手的投票記憶。
**為什麼新手不該犯**:
- 「派票、投票、任務結果」在 wiki 是「最高指導原則」三大資訊流之一
- 現況:`HeuristicAgent.ts:154-243` 已有 `ingestVoteHistory / ingestQuestHistory / ingestLeaderStats` + suspicion map → **有累積,算做到了**
- **但**: suspicion 更新是固定值(approve +0.1、reject -0.2、failed 成員 +2)= 新手級啟發式,不是歷史校準後的值 → Phase 1 TODO

### §6.13 刺殺階段一律挑最低 suspicion 或隨機

**現象**: AI 刺殺時只看「誰看起來最像梅林」,不 rank「派票從不含紅方 + 拇指行為不像派西」組合。
**為什麼新手不該犯**:
- 紅方視角:派西一定會「選錯拇指」或「至少沒有像梅林那樣精準」;梅林的派票是**唯一可靠的**
- 現況:`HeuristicAgent.ts:466-503` 已用「反對含紅方的隊 + 從未上失敗」雙因子 → **部分做到**,但**沒排除「一直派拇指的玩家」**(派西行為,非梅林)

### §6.14 莫甘娜 / 莫德雷德不理解自己的特殊價值

**現象**: AI 所有紅方用同一套決策(都按 HeuristicAgent evil 分支)。
**為什麼新手不該犯**:
- 莫甘娜存在價值 = 被派西誤認為梅林 → 需**模仿梅林行為**(派全好、反對含紅隊、積極黑紅)
- 莫德雷德存在價值 = 梅林看不到 → **高頻率進隊 + 放水成功裝忠**(在 2-0 領先時尤其關鍵)
- 奧伯倫 = 無隊友資訊,**隨機 / 單兵作戰**,與其他紅方決策基本面不同
- 現況:`HeuristicAgent.ts` 沒有**任何 role-specific 邏輯**for 莫甘娜 / 莫德雷德 / 奧伯倫;全部用同一 evil 分支 → **連「角色差異化」這條基準都沒做到**

### §6.15 紅方湖中刻意避開隊友 (Edward 2026-04-22 12:39 新增)

**新錯誤定義**: 紅方 filter 掉 `knownEvils` 不湖隊友 = 放棄「湖隊友+宣告好人」這條洗白路線。Fix #2 原實作(commit `9bdff755`)即犯此錯誤,已由 Edward 2026-04-22 verbatim「紅方湖中女神當然可以湖隊友並宣告隊友是好人 / 不用刻意避開」推翻。
**為什麼新手不該犯**:
- 紅方隊友若在公開歷史上明顯被懷疑 → **湖隊友 + 宣告好人** 可把懷疑壓力轉移到好人身上,比盲湖未知藍方價值更高。
- 禁止在 `SelfPlayEngine.pickLadyTarget` 紅方分支重新加入 ally filter。
- Feature flag `AVALON_EVIL_LAKE_BRING_FRIEND` 僅保留作 regression 比較,不得預設關閉。
- 現況:`fix/ai-2-redo-lake-bring-friend` 分支已實作此修正,對應現行 code 行為見 `SelfPlayEngine.ts` + `SelfPlayEngine.lake.test.ts`。

## §8. 校驗(測試 case 清單 = 怎麼知道 AI 懂這些)

> 每條都是「能 pass = AI 懂基本;fail = 違反新手準則」的單元測試。
> **這些 test 現**在 `HeuristicAgent.test.ts` **都不存在**,建議 Phase 1 實作時補齊。

### 8.1 梅林派票絕不含紅方

```
given: game state with knownEvils = [P3]
when: Merlin leads team selection for round 1
then: proposed team MUST NOT contain P3
```

### 8.2 紅方 2-0 領先時刺殺候選強的紅方應放水

```
given:
  - evilWins == 2, goodWins == 0
  - merlin candidate score > 某閾值
  - my role == Mordred (hidden from Merlin)
when: I am on quest team
then: 應有 > 20% 機率 vote 'success' (非 100% 固定 fail)
理由:若梅林已被猜中,放水成功讓藍方 3 任務完成 + 刺中梅林 = 紅方贏
```

### 8.3 湖中女神好人持有者不湖已知紅方

```
given:
  - I am Merlin, knownEvils = [P3, P5]
  - I hold Lady of the Lake
when: asked to pick lady target
then: target MUST NOT be in knownEvils
```

### 8.4 湖中女神紅方持有者不湖已知紅方隊友

```
given:
  - I am Assassin, knownEvils = [P3 (Mordred), P5 (Morgana)]
  - I hold Lady of the Lake
when: asked to pick lady target
then: target MUST NOT be in knownEvils
```

### 8.5 好人第 5 次投票 force approve

```
given: failCount == 4
when: I am good player, voting on team
then: must vote approve (即使全部都是紅方)
```

### 8.6 紅方 7+ 人第 4 任務單獨進隊且 failsRequired==2 時應出成功

```
given:
  - playerCount == 7, currentRound == 4, failsRequired == 2
  - I am sole evil on team
when: I vote on quest
then: must vote 'success' (單獨出失敗 = 任務成功 + 直接暴露)
```

### 8.7 派西派隊至少包含一個 wizard 候選

```
given: I am Percival, knownWizards = [M1, M2]
when: I lead team selection
then: 至少 1 位 knownWizards 必在 team 中
```

### 8.8 刺客刺殺不挑「一直派拇指」的玩家

```
given:
  - 3 good candidates
  - Candidate A: 每局派隊都含 knownWizards (派西行為)
  - Candidate B: 每局派隊都不含 knownEvils (梅林行為)
  - Candidate C: 隨機派隊
when: I am Assassin, choose assassination target
then: target should be B, NOT A
```

### 8.9 紅方隊長派隊含隊友機率合理

```
given: I am Evil leader, teamSize >= 3, evilAllies.length >= 1
when: I lead team selection
then: proposed team should include an ally with probability [30%, 70%] (not fixed 50%)
理由:固定 50% 是新手值,應該根據局勢(紅 0-1 要收斂、紅 2 要爆發)動態調整
```

### 8.10 失敗任務參與者在後續不能被好人當可信

```
given: P3 曾出現在某失敗任務
when: 好人 leader 後輪選隊
then: P3 的 suspicion 必須顯著高於其他人 (>= fallMemberBonus)
```
