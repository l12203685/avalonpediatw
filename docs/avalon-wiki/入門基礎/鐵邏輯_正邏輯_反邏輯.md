## 鐵邏輯
### 最重要的資訊參考依據
* 邏輯與資訊的參考依據
* 任務結果 > 湖中女神 > 派票 >= 黑白球 >= 發言 >= 狀態
    * 任務結果舉例: 1-5 158 OXX
        * 表示 158 "至少" 兩個紅方
        * 出現兩張失敗後, 必定認為另外兩位玩家為紅方
    * 湖中女神舉例: 0>3 O
        * 基於遊戲勝利目標, 藍方不會說謊
        * 湖中結果代表只有壞湖好, 好湖好, 壞湖壞
        * 雖然遊戲規則沒有不允許, 但基本上藍方湖紅方不會說對方是藍方陣營的
### 不單是明面上的直覺思考
* 是正確的邏輯, 絕對正確的事實
    * 要特別注意的是，所謂的絕對正確邏輯，基本上是依附在遊戲規則之下的
    * 所以這種絕對正確的邏輯也稱為是「鐵邏輯」
* 牌局範例
    ``` json
    158 oxx
    2349 ooox
    0 > 6 o
    ``` 
    * 158 至少兩壞, 2349 至少一壞
    * 670 最多一壞
    * 0 湖 6 好 => 6 必為好人

## 正邏輯與反邏輯
### 正邏輯
* 最大效益原則
* 明面上第一直覺的事情
* 更正確的說法是機率更大的假設 
* 建立在紅方所做出的行動皆是為了快速達到三紅
* 但記得正邏輯只是機率更大的假設，不是絕對。
    * 50 雙壞
        * 50雙尾家各出一局直接兩紅開局是最有效益的
        * 反推>>第一局藍燈第二局紅燈或第一局紅燈 第二局藍燈都表示50不是兩個相認壞
        * 範例牌局
            ``` json
            350 ooo
            2350 ooox
            0>5 o
            ```
        * 5家尾派場外三白被洗掉
            * 假設五家好人外三白會有一到二壞，大多數是兩壞
            * 假設五家壞人外三白會有一到兩好，大多數是兩好
            * 因為不太會兩壞洗兩壞
    * 行動方式
        * 出者恆出，藏者恆藏
        * 進行反推思考
        * 第一局有相認壞直接出失敗，輪到0家尾家出失敗，兩紅開局
        * 反推>>第一局紅燈 第二局藍燈 第三局照派紅燈 表示 原本就在第一局的玩家會偏好
        * 兩個相認壞沒必要送第二局藍
        * 牌局範例
            ``` json
            257 oox
            3790 oooo
            0>3 o
            3790 ooox
            ```
        * 7 偏好
    * 兩藍開局不會是相撞
        * 牌局範例
            ``` json
            350 ooo
            2350 oooo
            0>2 o
            2350 ooox
            2>7 o
            ```
        * 2如果是相認壞第二局會開紅
        * 2就算是壞也互不相認
        * 不是相撞 (350沒有相認兩壞)
        * 第一局相撞會分出階級, 第二局會有失敗
        * 2被0湖好 => 2 偏好
        * 代表 350 原本就有紅方
        * 50 不會兩壞 (基於最大效益原則)
        * 綜合湖中, 最合理的選擇是 27 + 350 選一到二 + 從 14689 選一到二
### 反邏輯
* 透過多數人會以正邏輯思考的路線, 刻意做出視角盲點或表面上低效益的行動
* 會特別指"表面上"是因為, 反邏輯行動的真實效益就發生場上的人都以正邏輯思考的後續發酵
## 結論
* 正邏輯不是絕對沒問題的但能應付百分之八十五的情況
* 紅方也有可能故意打戰術、運用正反邏輯去欺騙藍方雜魚, 這在老手局相當重要的一件事
* 但對於新手來說先學習正邏輯才是最應該要的基本功
