# 牌譜紀錄格式
* 這是目前研究與討論過, 最有效率也最精簡的紀錄格式
* 如果有其他更好或是更方便的紀錄格式也歡迎提供
* 歡迎提供牌譜: https://forms.gle/55hYJUQjs8vqujFR8

## 量化與質化
遊戲過程的描述構成可分成"量化"與"質化"的部分.
* 量化
    * 可以客觀描述的資訊與事實
    * 不會因為不同玩家做出不同的紀錄
* 質化
    * 主觀描述的資訊
    * 會因為不同玩家做出不同紀錄
* 這邊建議, 先將**量化**的內容紀錄好, 並能清楚解讀對應的資訊後, 再進入質化的討論. 
    * 原因在於, 每位玩家的主觀意識與感受並不相同
    * 覆盤跟討論牌局時若沒有相同的討論基準會容易失焦
    * 另外, 質化的內容不太能完整複製, 對於進步的幅度有限.

## 記錄要點與格式
如遊戲目標與流程中所提, 一場遊戲過程不外乎幾個階段, 完整的紀錄必須記錄每個階段的重點內容:
* **{隊伍組合選擇階段}: 每一位隊長選擇了何種隊伍組合**
* **{投票階段}: 對應隊長選出的隊伍組合, 每位玩家的贊成反對**
* **{任務階段}: 對應贊成過半或強制局的隊伍組合, 任務結果 (任務失敗數量)**
* **{湖中階段}: 第二, 三, 四局結束後, 湖女的使用對象與宣告看到的陣營**
* **{刺殺階段}: 壞人陣營的角色分別認為誰是梅林, 刺客最終選擇指認哪位玩家為梅林**
* **{角色分配}: 帶入腳色回推當時每位玩家的視角.**

## 記錄格式
``` json
{隊伍組合} {異常票} 
({任務結果})
({湖女與湖中結果})
{各角色位置分配}
{各角色刺殺對象}
```

## 說明
* 數字代表該次任務的組合, 空格後代表特別票 (-表內黑/+表外白)
    * 158 8* 0+ 代表該輪隊長選擇 158, 並且 8 家開了黑球, 0 家開了白球
* 換行代表輪到下一個階段 (派票/出任務/湖中)
* o/x 或 O/X 表示任務成功與失敗的數目
    * ooxx 代表兩張任務失敗, OOX 代表一張任務失敗
* 湖中結果以 x>y o/x O/X 代表
    * 0>5 X 代表 0 湖 5 說壞人/紅方/邪惡
    * 5>7 O 代表 5 湖 7 說好人/藍方/正義
* 最後六個數字為能力角色, 刺客, 莫甘娜, 莫德雷德, 奧伯倫, 派西維爾, 梅林的位置
    * 例如 305791 代表 刺娜德奧派梅 位置在 305791
* 後四個數字為紅方各自指認的刺殺對象
    * 例如 1112 代表 刺娜德奧 指認的梅林分別為 1112
## 阿瓦隆的遊戲組合數
* 阿瓦隆光是角色 x 位置的排序就有 15 萬種組合
* 假設是同樣10位玩家, 不同玩家在不同位置則有 360 萬種組合
* 所有總組合數共有 5400 億
* 這些數字只是要表達一件事: 遊戲場數永遠不夠多
