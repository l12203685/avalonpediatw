## 視角
* 視角通常用以描述該玩家對場上其他玩家陣營的看法
* 代表一個人的視角與其認定的區間

### 視角區間範例
* 常見的區間的描述方式如下: 
    * 8 人局範例
        ``` json
        145
        257
        368
        456
        358
        oox
        1567
        1278
        1278
        1456
        1267
        ooox
        8>7 o
        ```
        * 3 家的視角是 58 至少一壞, 1267 至少一壞, 4 是未知
        * 8 家的視角是 7 鐵好, 35 至少一壞, 126 至少一壞, 4 是未知
    * 10 人局範例
        ``` json
        158 8- 0+
        234 8+
        369
        147
        159
        oxx 
        3690
        1478
        3489
        3469 8+
        3680 
        ooox
        0>4 X
        ```
        * 場上的共同視角是 04 至少一壞, 159 至少兩壞, 3680 至少一壞, 247 最多一壞
        * 0 家視角是 4 鐵壞, 159 至少兩壞, 368 至少一壞 => 27兩好, 且 159兩壞 368一壞

## 區間派票法
### 什麼是區間
* 區間通常代表某幾位玩家的壞人區間分割
* 切分區域: 場上總共有 10 位玩家繞著圓桌而坐 (記得嗎我們是圓桌武士)
* 就像切蛋糕, 透過派票把圓桌切成三個區域

### 理由
* 阿瓦隆一個人平均只有2.5次的派票次數
* 只有三張票單純給好人的時候資訊會過少，所採用此方法
* 通常運用在 1-1~1-4 梅林能快速給出壞人和好人的區域
* 認為大多數時候好人、壞人會亂數分配位置，大多數時候不會連坐
* 範例 (1梅林)
    *  1-1 派 158: 這個派票把區域切成 [2, 3, 4], [6, 7], [9, 0]
    * 因為梅林看到的拇指落在這三個區域

## 邏輯互斥的區間
* 1-1 ~ 1-5 可以分成幾類
    * 被閃開的是紅方 (至少一壞)
    * 有紅方疊的是紅方隊友 (至少二壞)
    * 少數被派過後不疊 (至少一壞)
    * 剩下就是六選三
* 就算全部都給一個容錯
    * 被閃開的是紅方 (一壞一容錯)
    * 有紅方疊的是紅方隊友 (二壞一容錯)
    * 少數被派過後不疊 (一壞一容錯)
    * 加起來就是 2 + 3 + 2 = 7 
    * 剛好剩下兩個可以選擇 + 自己 = 三個人
* 1ab, 2cd 的情況
    * 如果認為 1 & 2 都是相認壞, 不可能 1 & 2 "都" 帶隊友
    * 相認壞只有三人, 至少有一人的視角是"沒帶隊友" = "閃隊友"
