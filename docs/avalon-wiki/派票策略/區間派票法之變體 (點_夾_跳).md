# 點拇指/跳拇指/夾拇指
* 直接指出拇指的位置
    * 派拇指的前一位玩家或後一位玩家 (派票會塞在拇指中間)
    * 在 1-1 ~ 1-4, 透過派 5 表示 0 偏壞, 派 0 表示 5 偏壞
* 順著派出去但跳過一個拇指, 再做出區間
    * 459 => 跳6 給9點80
    * 238 => 跳4 給8點79
* 夾拇指的位置
    * 壓縮區間的極致就是夾人
        * 也是大多數人表達誰是壞人最常用且最直接的方法
        * 我派你左右的玩家，就是不想派你!
    * 379 => 夾8 給 [0, 1, 2], [4, 5, 6] 兩個區間
    * 139 => 夾2, 0 
* 範例 
    * 3 梅林派370 (點尾派 + 區間)
        * 區間為 [4, 5, 6], [8, 9], [1, 2]
        * 在派了 0 的狀況下, 代表"點"5是壞人
        * 結合區間, 可以得到 [5, 8, 9] 兩壞, [1, 2] 一壞, [6] 偏好這個結論
        * 4 不一定則是因為派了 [3] 也有可能是在點 [4]
    * 3 梅林派347 (跳 5 + 區間)
        * 跳過了5 (高機率是5, 理由是5好人3家梅林可以直接345)
    * 1 梅林派127 (跳 3 + 塞7點68)
        * 跳過了3 (高機率是3, 理由是3好人1家梅林可以直接123)
        * 68 至少一壞, 所以梅林想要派中間的 7
    * 1 梅林派189: 就是表達0拇指
    * 1 梅林派568: 
        * 第一層: 568 都沒拇指
        * 第二層: 區間是 [2, 3, 4], [7], [9, 0], 7 偏特別壞 (68夾7)
    * 1 梅林派134: 
        * 表示 134 都沒有拇指
        * 同時也表達 2 家是拇指
        * 可以同時獲得 134 好人 2 壞人雙重資訊
    * 1 梅林派135: 
        * 夾了 2 和 4 
        * 表示 2 或 4 最少有一壞以上 

# 切分區域
* 除了點、夾、跳以外，也有前置位、後置位判斷的方式
* 前置位: 1 ～ 4 
* 後置位: 6 ～ 9
* 50獨立看
* 4梅林且派票如下:
``` json
378 1+
279
590 3+
145

* 在點夾跳中，代表145沒拇指、{2, 3} 開壞人
* 切分區域的思考上，表示後置位6789可能開2根以上的拇指
* 以本場配置而言，實際上梅林的三拇指在789
```

# 點跳夾派法的優缺點與比較
|  | 區間派票法 | 一般派票 |
| --- | --- | --- |
| 優點     | 資訊多 | 資訊少 |
| 缺點 | 死亡率高 + 易被發現能力 | 死亡率低 |
| 上手度 | 難 | 易     |
| 接受度 | 主觀 | 客觀 |
